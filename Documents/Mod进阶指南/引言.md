# Mod进阶指南

本指南面向已经能够编写基础职业 Mod 的开发者，重点解释两个问题：
- 如何理解技能 DSL 编译后的执行链
- 如何通过动态判定规则修改回合流程

建议阅读顺序：
- [判定实现](./判定实现.md)
- [DSL与规则链接](./DSL与规则链接.md)

如果你还没有写过最基本的职业 Mod，请先阅读：
- [Mod基础指南](../Mod基础指南/引言.md)

## 这一部分解决什么问题

基础指南足够让你写出：
- 普通攻击
- 防御
- 资源变化
- 简单的被动技能

但当你想实现下面这些能力时，就需要理解进阶机制：
- 临时改写某个判定阶段
- 让一个技能在“下一回合”才开始生效
- 在特定阶段插入一个额外规则
- 实现转移、反击、延迟保护、阶段性强制处理等复杂效果

这些能力主要依赖：
- `DSLforSkillLogic.SourceFile`
- `Mutation`
- `DynamicJudgeRulePool`
- `JudgeRuleManager`
- `DynamicJudgeRuleName`

## 你会看到的核心结论

1. DSL 不是直接结算，而是先构造 `SourceFile`。
2. `SourceFile.Compile()` 会把技能变成 `Intent`。
3. 如果技能里使用了 `LinkJudgeRuleDynamic(...)`，那么编译阶段还会把动态规则注册到裁判系统。
4. 真正执行一回合时，`Judger` 会按多个 `JudgeStage` 依次运行基础规则、覆盖规则和前后修饰器。
